Prosperity Bank

Project overview

Design problem

My goal for this project is to design a simulated banking application. The objective is to implement basic functionality of any banking system such as deposit and withdrawal, a scheduled pay system and a separate admin page. This project is to develop my skills as a developer by learning a new technology and utilising what I've learnt in my bachelor studies.

System architecture

A system architecture was created to ensure a vision is created for the project, thus can be used to review the overall state of the project during development. Furthermore, this allows a smoother transition in future development by reviewing this project’s vision, while also allowing for quicker changes in the system as it provides a clear picture for the project. 

                        Prosperity bank web pages: Presentation Layer ( User interface ) - Contain all the view pages

                        Layer 2: Business logic layer - All functionality is presented in this layer
                            Bank - this will handle all customer and transaction information 
                            
                            BillPay - this will handle the scheduled payment functionality 
                            
                            Security - this will handle the user authentication

                            Other - any extra functionality 
                            
                            Data Access - This will provide all the functionality one needs to connect to database and retrieve information 

                        Prosperity bank database - Local database using Microsoft SQL server management

Entity relationship diagram

An entity relationship diagram was created to provide a better graphical representation of our database relationship. As we would be using “code first approach”, this diagram helps us map out our model’s attributes and relationship between entities, thus minimising redundancy.

Reflection

What went well

Frequent commitment to the project while working full time, and successfully completing the project. Able to learn a new technology and further demonstrate my knowledge from past projects.

What could have been improved

As the focus for this project was the functionality there was a lack of focus towards the design of the project. Therefore, the web pages are very simple and bland. Furthermore, some functionality took much longer than planned, due to the lack of knowledge, which could have been avoided from better research and practice.

What I would add to the project in the future

  • - Unit testing
  • - Creating new bank account
  • - Loan payment
  • - Interest
  • - Better web design
